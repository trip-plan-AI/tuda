## 🎯 Общая информация

Этот документ является живым руководством по проекту `Travel Planner Monorepo`.
Здесь будут описываться архитектурные решения, стек, пошаговые инструкции и прогресс работы.

### 🏛️ Архитектура и Инфраструктура

- **Repo**: Turborepo (Monorepo) + pnpm
- **Хостинг**: VPS/Cloud (Backend)

### 💻 Frontend

- **Core**: Next.js (App Router)
- **UI Framework**: Tailwind CSS + shadcn/ui
- **State Manager**: Zustand
- **Maps**: Leaflet / React-Leaflet + Mapbox API

### ⚙️ Backend

- **Core**: Nest.js
- **Database**: PostgreSQL
- **ORM**: Drizzle ORM
- **Geo**: Turf.js / PostGIS

### ✅ Основная цель Monorepo

Единая типизация (TypeScript) между бэкендом и фронтендом для обеспечения целостности данных и удобства разработки.

## 📝 Заметки по менторингу

Здесь будут храниться детальные пошаговые объяснения и команды.

### 🚀 Продолжаем работу над `Travel Planner Monorepo`

Мы успешно инициализировали все базовые приложения и пакеты. Теперь нам нужно убедиться, что `pnpm` корректно управляет всеми этими частями. Для этого нужно настроить `pnpm-workspace.yaml` и установить зависимости.

### 📝 Шаг 8: Настройка `pnpm-workspace.yaml` для включения всех `apps` и `packages`

Файл `pnpm-workspace.yaml` сообщает pnpm, где искать пакеты и приложения в вашем `monorepo`. Turborepo уже создал его, но нам нужно убедиться, что он включает все наши новые приложения.

**Что делают эти команды:**

- `cd ../..`: Возвращает вас в корень `monorepo` (`/home/dmitriy/projects/trip/travel-planner`), если вы сейчас находитесь в `packages/ui`. Это необходимо перед чтением `pnpm-workspace.yaml`.
- `read_file pnpm-workspace.yaml`: Считывает содержимое файла `pnpm-workspace.yaml`.

**Зачем это нужно:**

- Чтобы я мог проверить текущую конфигурацию рабочих областей и убедиться, что она охватывает все только что созданные папки (`apps/web`, `apps/api`, `packages/types`, `packages/ui`).

**Как проверить успех:**

- После выполнения команды вы увидите текущее содержимое `pnpm-workspace.yaml`.

---

**Пожалуйста, выполните следующие команды в вашем терминале WSL, начиная с текущей директории:**

```bash
cd ../..
read_file pnpm-workspace.yaml
```
