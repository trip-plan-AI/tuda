## 🔑 Ключевые находки

### 1. **Проблема 1: Недостаток POI для некоторых городов**
- Kyoto: 1 POI (должно быть 20+)
- Москва: 1 POI (должно быть 20+)
- **Причина:** KudaGo вернул минимум, Overpass API не сработал
- **Решение:** Проверить конфигурацию provider search или использовать fallback

### 2. **Низкое согласие на иностранные города (20-50%)**

Города с низким согласием:
- Chiang Mai: 20%
- Tirana: 40%
- Sofia: 40%
- Hanoi: 40%
- Valencia: 40%
- Kraków: 50%

**Вывод:** Оба модели выбирают правильные **категории**, но разные конкретные **места**.

### 3. **Высокое согласие на культурные запросы (70%+)**

Города с высоким согласием:
- Tbilisi: 73%
- Split: 78%
- Bangkok: 67%

**Вывод:** Когда категория четко определена (музеи, еда, архитектура), согласие выше.

### 4. **OpenRouter выбирает больше POI**
- Yandex avg: 5.9 POI
- OpenRouter avg: 6.8 POI (на 15% больше)
- OpenRouter более "щедрый" в выборе

### 5. **Yandex быстрее**
- Yandex avg: 3,087ms
- OpenRouter avg: 3,581ms
- Yandex на ~16% быстрее

### 6. **Бюджет игнорируется обоими**
- Запросы с явным бюджетом (10k, 15k, 18k, 20k руб) - оба модели не учитывают
- Предполагают, что бюджет обрабатывается на других этапах pipeline

## 🎯 Рекомендации по гибридной стратегии

### Вариант 1: По типу запроса (РЕКОМЕНДУЕТСЯ)

```typescript
if (input.text.includes("культур") || input.text.includes("музе")) {
  // Tbilisi (73%), St.Petersburg (60%) → хорошо
  useProvider = 'openrouter'; // OpenRouter выбирает только музеи
} else if (input.text.includes("еда") || input.text.includes("паэлья")) {
  // Valencia, Split → хорошо
  useProvider = 'yandex'; // Yandex более точен в еде
} else if (input.text.includes("бюджет") || /\d+\s*(?:к|тыс|тысяч)/.test(input.text)) {
  // Budget-conscious
  useProvider = 'yandex'; // Yandex немного лучше
} else {
  // Default
  useProvider = 'yandex';
}
```

### Вариант 2: Ensemble для низкого согласия

Если две модели выбирают разные места (Chiang Mai 20%, Valencia 40%):
- Запустить обе параллельно
- Объединить результаты, отсортировать по рейтингу
- Выбрать top-6
- Стоимость: +2x API, но качество выше

### Вариант 3: Качество > Скорость

Для важных запросов (первый раз пользователя, долгая поездка):
- Запустить обе модели параллельно
- Объединить, дедупликировать по названию
- Вернуть лучший из двух (по рейтингу)
