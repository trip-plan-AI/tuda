# Тесты сравнения YandexGPT vs OpenRouter

Модуль для тестирования семантического фильтра poi на YandexGPT и OpenRouter (GPT-4o-mini).

## Структура

```
tests/ai/provider-comparison/
├── data/
│   └── test_queries.json          # 22 тест-сценария (русскоязычные запросы пользователей)
├── results/
│   └── text_queries_*.json        # JSON результаты каждого теста
├── run_text_queries.py            # Тест-раннер (основной)
├── run_tests.py                   # Альтернативный раннер (для структурированных тестов)
├── README.md                      # Этот файл
├── VIBE_BUDGET_TEST_RESULTS.md    # Анализ первых 7 тестов
├── FINAL_ANALYSIS_RUSSIAN_QUERIES.md  # Анализ первых 12 тестов
└── COMPREHENSIVE_RESULTS.md       # Финальный отчёт (22 теста)
```

## Запуск тестов

### Базовый запуск
```bash
python3 run_text_queries.py
```

### С кастомным API URL
```bash
python3 run_text_queries.py --api-url http://localhost:3001
```

## Результаты (22 теста)

```
Total POI:       Yandex: 106  |  OpenRouter: 117
Avg POI/test:    Yandex: 5.0  |  OpenRouter: 5.6
Avg time:        Yandex: 2,800ms  |  OpenRouter: 3,400ms (+21%)
```

## Ключевые выводы

✅ **YandexGPT:**
- Быстрее (2,800 vs 3,400ms)
- Более консервативен в выборе (5.0 vs 5.6 POI)
- Лучше для бюджетных запросов

✅ **OpenRouter:**
- Выбирает больше POI (+10%)
- Лучше для культурных/исторических запросов
- Более "щедрый" в разнообразии

⚠️ **Проблемы:**
- Некоторые города возвращают мало POI (Tirana, Kyoto, Brasov: 1 POI)
- Причина: KudaGo не покрывает иностранные города, Overpass API не работает
- Нужна диагностика provider-search pipeline

## Рекомендации по внедрению

Выбор провайдера на основе:
1. **Тип запроса:** культура/история → OpenRouter, вайб/бюджет → YandexGPT
2. **Язык города:** русский город → YandexGPT, иностранный → может быть оба
3. **Режим питания:** gastrotour → OpenRouter (выбирает больше food)

```typescript
// Псевдокод
if (query.includes("культур") || query.includes("исторі")) {
  provider = 'openrouter'; // Лучше для истории
} else if (query.includes("бюджет") || /\d+.*(?:k|тыс)/.test(query)) {
  provider = 'yandex'; // Лучше для бюджета
} else {
  provider = 'yandex'; // Default
}
```
